
# Bridged(桥接模式)

    占用区域网ip，与宿主主机再同一网段内，例如
    主机： 192.168.0.133
    虚拟机： 192.168.0.100
    会占用区域网 ip资源，能够访问外网vmnet0
    
# NAT(网络地址转换模式)

    不占用区域网ip，使用vmnet8，在宿主主机和虚拟机之间组成新的区域网，例如
    主机：
        vmnet0 192.168.0.23  【本机wifi】
        vmnet8 192.168.56.*  【自动分配ip】  
    虚拟机：
        192.168.56.100 【通过vmnet8网卡（主机）访问外部网络】 
        通过做端口映射可以，远程连接虚拟机
            映射关系 192.168.0.23:9999 映射到 192.168.56.100:22 【虚拟机和区域网其他机器.docx】
            然后就可以通过其他区域网的机器 ssh远程连接虚拟机 - 192.168.0.23:9999 
    虚拟机可以访问外部网络，通过vmnet8
    
# Host-only(主机模式) 。
    
    虚拟系统和真实系统之间完全无法相互访问，但是虚拟系统之间能相互访问
